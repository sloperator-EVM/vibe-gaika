# Пример бота для GAICA

Этот архив показывает рабочую базовую структуру бота под актуальный socket API GAICA.

Что внутри:

- `main.py` — точка входа
- `gaica_bot/models.py` — OOP-модели сообщений и состояния игры
- `gaica_bot/client.py` — сетевой клиент и цикл обработки сообщений
- `gaica_bot/sample_bot.py` — простой бот-пример

Что умеет пример:

- подключаться к игре по `host:port`
- разбирать `hello`, `round_start`, `tick`, `round_end`
- хранить карту, серию, себя, противника и `snapshot` в виде dataclass-объектов
- отправлять команды в формате API

Что он не делает:

- не пытается играть сильно
- не использует pathfinding
- не анализирует линии прострела и препятствия глубоко
