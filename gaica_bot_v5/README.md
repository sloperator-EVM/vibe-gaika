# Пример бота для GAICA

Этот архив показывает рабочую базовую структуру бота под актуальный socket API GAICA.

Что внутри:

- `main.py` — точка входа
- `gaica_bot/models.py` — OOP-модели сообщений и состояния игры
- `gaica_bot/grid.py` — быстрая клеточная карта поверх `floor` и текущего `snapshot`
- `gaica_bot/client.py` — сетевой клиент и цикл обработки сообщений
- `gaica_bot/sample_bot.py` — простой бот-пример

Что умеет пример:

- подключаться к игре по `host:port`
- сразу после подключения отправлять обязательный `register`
- разбирать `hello`, `round_start`, `tick`, `round_end`
- хранить карту, серию, себя, противника и `snapshot` в виде dataclass-объектов
- разбирать `round_start.level.floor`, `player_spawns` и `static_obstacles`
- использовать `state.current_obstacles` как полный актуальный obstacle-state на тике
- строить клеточную карту на каждом тике с переиспользованием кэша уровня из `round_start`
- использовать `numpy` для масок и overlay-слоёв, если он доступен в окружении
- разбирать в `snapshot` скорость игрока (`velocity`) и скорость снарядов
- работать с новым `tick.snapshot` без `level/effects/debris`
- одинаково обрабатывать `round_end.result` на любом раунде серии, включая последний
- отправлять команды в формате API

Что он не делает:

- не пытается играть сильно
- не использует pathfinding
- не анализирует линии прострела и препятствия глубоко
